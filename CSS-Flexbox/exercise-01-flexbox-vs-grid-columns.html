üí° Key Differences:
  • Grid: Define all columns at once with grid-template-columns

  • Grid Advantage: More declarative, easier to read
  • Flexbox Advantage: More control per item, can use flex-basis for responsive behavior

1.2 Proportional/Fractional Layouts

Concept: Dividing space proportionally (like 1:2:1 ratio).

üî∑ CSS Grid Approach

grid-template-columns: 1fr 2fr 1fr;
/* Space divided in 1:2:1 ratio */

2 (2x wider)

üî∂ Flexbox Approach

display: flex;
/* Use flex property on items */
.item1 { flex: 1; } /* 1 part */
.item2 { flex: 2; } /* 2 parts */
.item3 { flex: 1; } /* 1 part */
1
2 (2x wider)
3
üí° Mental Model:
  • Grid: 1fr 2fr 1fr = "Divide space into 4 parts, give 1, 2, 1"
  • Flexbox: flex: 1, flex: 2, flex: 1 = Same concept!
  • Both work identically for proportional layouts
  • Grid: More concise for container-level control
  • Flexbox: More flexible for item-level control

1.3 Mixed Units (Fixed + Flexible)

Concept: Sidebar with fixed width + flexible main content.

üî∑ CSS Grid Approach

grid-template-columns: 250px 1fr;
/* Sidebar fixed, content flexible */
Sidebar (250px)
Main Content (flexible)

üî∂ Flexbox Approach

display: flex;
.sidebar { width: 250px; flex-shrink: 0; }
.content { flex: 1; } /* Takes remaining */
Sidebar (250px)
Main Content (flexible)
üí° Real-World Use: This is the classic sidebar + content layout. Both Grid and Flexbox handle it well, but:
  • Grid: More semantic - the layout is defined at container level
  • Flexbox: More flexible - can easily add/remove items, change order
  • Both: Work perfectly for this pattern!

üéØ General Rule:
  • Use Grid when you need to control both rows AND columns, or when you want explicit column definitions
  • Use Flexbox when you have a single row/column of items that need flexible sizing
  • Use Both Together: Grid for page layout, Flexbox for component alignment (very common!)
