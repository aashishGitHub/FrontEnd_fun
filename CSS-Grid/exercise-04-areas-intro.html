4.1 Basic Named Areas

Concept: `grid-template-areas` lets you create a visual map of your layout using named areas. This is like drawing your layout!

grid-template-areas: "header header" /* Row 1: header spans 2 columns */ "sidebar content"; /* Row 2: sidebar | content */ /* Then assign elements to areas: */ .header { grid-area: header; } .sidebar { grid-area: sidebar; } .content { grid-area: content; }
Header
Sidebar
Content
üìê How It Works:
  • Each string in quotes = one row
  • Each word in the string = one column
  • Same name repeated = that area spans multiple columns/rows
  • Use `grid-area: name` to assign elements to areas

4.2 Empty Cells with Dots (.)

Concept: Use "." (dot) to create empty cells. This lets you center content or create gaps in your layout.

grid-template-areas: "header header header" /* Header spans all 3 columns */ ". content ."; /* Content in middle, empty on sides */
Header (spans all)
Content (centered)
üí° Pro Tip: The dots create empty cells. You can use ".", "..", or "..." - they all mean the same thing (empty cell). This is useful for centering content!

4.3 Classic Website Layout

Concept: This is the most common layout pattern! Header, sidebar, content, and footer - all defined with named areas.

grid-template-columns: 200px 1fr; /* Sidebar | Content */ grid-template-rows: 60px 1fr 40px; /* Header | Main | Footer */ grid-template-areas: "header header" /* Header spans both columns */ "sidebar content" /* Sidebar | Content */ "footer footer"; /* Footer spans both columns */

üåê Real-World Use: This exact pattern is used by millions of websites! The beauty of named areas is that you can rearrange the layout just by changing the strings - no need to recalculate grid positions.

4.4 Complex Area Layout

Concept: Use descriptive names for your areas. This makes your CSS self-documenting and easier to understand.

grid-template-areas: "nav nav nav" /* Navigation spans all */ "article1 article2 sidebar" /* Three columns */ "article3 article3 sidebar"; /* Article3 spans 2 cols */

üí° Mental Model: Notice how "article3" appears twice in the last row? That makes it span 2 columns. The "sidebar" appears in both row 2 and row 3, so it spans 2 rows automatically!

4.5 Responsive Areas Pattern

Concept: The power of named areas shines in responsive design! Just rearrange the strings in media queries.

/* Mobile: stack everything */ grid-template-areas: "header header header" "sidebar sidebar sidebar" "content content content" "footer footer footer"; /* Desktop: rearrange with media query */ @media (min-width: 768px) { grid-template-areas: "header header header" "sidebar content content" "sidebar content content" "footer footer footer"; }

üîÑ Try This: Resize your browser window! On mobile (narrow), everything stacks. On desktop (wide), the sidebar moves to the left. This is responsive design made easy!
üéØ Key Advantage: With named areas, you don't need to recalculate grid positions. Just rearrange the strings! This is much easier than using grid-column and grid-row.
