5.1 Overlapping Areas

Concept: Areas can overlap! This means TWO DIFFERENT elements can be assigned to areas that share the SAME grid cells, creating layered effects.

üîë Key Understanding:
  • Regular Areas: Each cell belongs to ONE area only
  • Overlapping Areas: Multiple elements can occupy the SAME cells
  • How it works: Assign different elements to the same area name, or use positioning
  • Visual effect: Elements layer on top of each other (like stacking cards)
  • Control: Use z-index to control which appears on top

Example 1: Visual Layout (NOT True Overlap)

Note: This example shows a layout, but these areas don't actually overlap - they're in different rows!

grid-template-areas:
  "image image" /* Row 1: image in cells (1,1) and (1,2) */
  "overlay content"; /* Row 2: overlay in cell (2,1), content in cell (2,2) */
/* These are SEPARATE areas in different rows - no overlap! */

‚ö†Ô∏è Important: This is NOT true overlapping! The "image" area is in row 1, and "overlay" is in row 2. They're in different rows, so they don't share the same cells. True overlapping requires elements to use the SAME area name (see Example 2 below).

Example 2: TRUE Overlapping - Same Area Name

This is REAL overlapping: Two different elements assigned to the SAME area name, so they occupy the SAME grid cells and layer on top of each other!

grid-template-areas:
  "hero"; /* Only ONE area defined */

/* But TWO elements use the same area! */
.hero-image { grid-area: hero; }
.overlay-text { grid-area: hero; } /* SAME area! */

Text Overlay (same area, on top!)
üí° How Overlapping Works:
  • Step 1: Define areas in grid-template-areas
  • Step 2: Assign multiple elements to the same area using grid-area: name
  • Step 3: Elements will occupy the SAME grid cells
  • Step 4: They layer on top of each other (like stacking)
  • Step 5: Use z-index to control which appears on top

  • Banners: Background + text/buttons
  • Image Galleries: Image + hover overlay

5.2 Asymmetric Magazine Layout

Concept: Real-world layouts are rarely symmetric. Use named areas to create visual hierarchy and interest.

grid-template-areas: "feature feature article1 sidebar" "feature feature article2 sidebar" "article3 article4 article5 sidebar"; /* Feature spans 2x2, sidebar spans 3 rows */

5.5 Fully Responsive Navigation Layout

Concept: The ultimate power of named areas: completely transform layouts at different breakpoints with just string changes!

/* Mobile: Stack everything */ grid-template-areas: "header" "nav" "main" "footer"; /* Tablet: Nav to side */ @media (min-width: 768px) { grid-template-areas: "header header" "nav main" "footer footer"; } /* Desktop: Add sidebar */ @media (min-width: 1024px) { grid-template-areas: "header header header" "nav main sidebar" "footer footer footer"; }
